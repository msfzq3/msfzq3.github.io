CFA三级考点笔记（十一）：交易执行、业绩评估及基金经理选择

发表于2022-01-18|更新于2023-02-19|CFALevel3
|字数总计:5.7k|阅读时长:18分钟|阅读量:

交易执行、业绩评估及基金经理选择部分在考试总成绩中的占比约为未确定

第34章：交易策略与执行

1.交易动机

  1. 追求利润（Profit-seeking）：被动投资跟踪基准；主动投资获取超额收益（Alpha）。

    • 亮池（Lit Market）：指交易透明度高，提供全方位交易数据的市场，如交易所。
    • 暗池（Dark Pool）：用于大额委托隐蔽交易意图，需要与冲销委托（Offsetting Orders）进行匹配。

      注意：暗池并非完全隐藏交易，而是隐藏交易前信息（Pre-trade），比如委托单。交易完成后仍会进行披露

    • 超额收益衰减（Alpha Decay）：由于投资策略被效仿，短期的超额收益可能被侵蚀衰减。
    • 交易紧迫度（Trade Urgency）：完成交易的紧迫程度，超额收益衰减速度越快，则交易紧迫性越高。

      执行风险（Execution Risk）：由于交易耗时，股票价格朝不利方向移动的风险。交易用时越长，执行风险越高。
      交易紧迫度越高，执行风险（Execution Risk）反而越低，因为交易用时较短，期间面临的市场波动风险较低。

  2. 风险管理与对冲需求（Risk Management/Hedge Needs）：再平衡（Rebalance），债券-久期/股票-beta/对冲基金-波动性。

  3. 现金流需求（Cash Flow Needs）：现金拖累（Cash Drag）：持有过多现金导致投资收益降低，可使用期货或ETF增大市场敞口。

  4. 公司行为（Corporate Action）、指数重建（Index Reconstitutions）、维保（Margin Call）

2.策略与选择

  1. 交易策略参数
    1. 交易订单特征：交易方向；绝对交易量；相对交易量（占日均成交量ADV的比例）
    2. 证券特征：证券类型；短期Alpha（Trade Alpha）；价格波动性/执行风险；买卖价差
    3. 市场状况：市场流动性（禁止做空制度）；证券流动性（新发行流动性较高）
    4. 风险厌恶：风险厌恶程度越高，则越希望降低风险敞口，采用快速交易方式
  2. 参考价格/价格基准（Reference Price/Price Benchmarks）
    1. 交易前基准价（Pre-trade）：在交易开始之前已确定的价格。
      例如：决策价格（Decision Price）；昨收盘（Pre-close）；今开盘（Open）；到达价格（Arrival Price）

      到达价格（Arrival Price）：订单到达市场的价格，Alpha交易/最小化交易成本时常用基准

    2. 日间基准价（Intraday）：在交易过程中产生且可能不断变化的价格。
      例如：交易量加权平均价（VWAP）；时间加权平均价（TWAP）
    3. 交易后基准价（Post-trade）：在交易结束后确定的价格。
      例如：当日收盘价（Close）
    4. 目标基准价（Price Target）：若发现股价被低估/高估，可以采用目标价为基准。
      例如：公允价格（Fair Value Price）
  3. 交易策略（Strategy）
    策略目标 紧迫度 价格基准 交易执行
    短期Alpha 高 到达价格 算法交易
    长期Alpha 低 不确定 拆单交易
    风险再平衡（Risk Rebalance），渐进对冲 低 TWAP 算法交易
    赎回-现金流驱动（Client Redemption） 高 收盘价 收盘集合竞价交易
    新指令-现金流驱动（New Trade Mandate） 低 收盘价 购买指数期货

3.策略执行（Execution）

  1. 交易执行选择
    1. 高附加值交易（High-touch Approaches）：由人工主导的交易行为，适用于大额、高紧迫度、低流动性的交易
      • 自营/做市交易（Principal/Market Maker Trades）：主要用于流动性较差的证券，可能采用询价交易（Request For Quote，RFQ）。
      • 代客交易（Agency Trades）：主要用于流动性较好的证券，由经纪商（Broker）在市场上寻找合适的订单，经纪商作为中介不参与交易。

      报价驱动（Quote-driven）：即做市商制度；指令驱动（Order-driven）：即竞价交易制度
      除了股票、ETFs及场内衍生品外，大部分交易都采用做市商制度

    2. 电子化交易：由电脑主导的交易行为，一般用于高流动性股票的小额交易。
      • 直接市场介入（Direct Market Access，DMA）：交易员通过经纪商的交易平台自主管理交易，通常用于小额场内/外汇交易。
      • 执行算法（Execution Algorithms）：通过特定算法进行交易。

      对于交易量大、流动性好的证券，人工交易可能发生抢跑现象（Front Running），即利用他人的交易行为提前布局获利
      因此，对于这类证券，使用电子化交易是更好的选择

  2. 算法交易（Algorithmic）：按照目标划分为执行算法（Execution）和逐利算法（Profit-seeking），本章仅介绍执行算法。
    1. 预设型算法（Scheduled）：适用于交易量偏低、流动性强、非紧急的订单。
      • 比例成交（Percentage of Volume，POV）：又称参与算法（Participation），根据市场成交量下单，不能保证成交；
      • 成交量加权（VWAP）/时间加权（TWAP）：在预设的时间段内按比例或均匀拆单，可以保证成交，但无法利用市场流动性。
    2. 流动性追踪（Liquidity-seeking）：又称机会算法（Opportunistic），多渠道（Multiple Venues）报价，适用于缺乏流动性的市场。
    3. 到价/前置/价差算法（Arrival Price/Front-loaded/IS）：跟随市场价格，用于高紧迫度交易，可降低执行风险，获取短期Alpha。
    4. 暗池交易（Dark Pool）：场外匿名配对大宗交易，可避免信息泄露，但不确定性较高（Less Certainty），无法确保完成交易。
    5. 比价系统（Smart Order Router，SOR）：多渠道报价及成交概率比对，寻找最优执行渠道，适用于小订单量的限价交易。

    低紧迫度/高流动性用预设，强冲击/低流动性用机会，中小规模/高紧迫度用到价，隐藏信息/非必须用暗池。

  3. 市场间对比
    1. 高透明度、强监管：股票市场、场内衍生品市场；
    2. 低透明度、低价格发现：债券市场、场外衍生品市场；
    3. 几乎无监管：外汇现货市场、跨国银行间市场（Interbank Market）。

4.交易评估（重点）

  1. 交易成本计量：执行价差（Implementation Shortfall，IS）
    1. 计算方法一：事后交易成本（Ex-post Trade Cost）
      • 执行价差IS = 模拟收益Paper Return - 实盘收益Actual Return
      • 模拟收益Paper Return = （当前价格Pn - 决策价格Ps） * 下单量S
      • 实盘收益Actual Return = 当前价Pn * Σ执行量Sj - Σ（执行价Pj*执行量Sj） - 交易费用Fees
    2. 计算方法二：执行价差的构成
      • 执行价差IS = 执行成本Execution Cost + 机会成本Opportunity Cost + 固定费用Fixed Fees
      • 执行成本Execution Cost = （交易价格Ptrade - 决策价格Porder） * 成交股数Sfilled = 延迟成本Delay Cost + 交易成本Trading Cost
        • 延迟成本Delay Cost = （到达价格Parrival - 决策价格Porder） * 成交股数Sfilled
        • 交易成本Trading Cost = （交易价格Ptrade - 到达价格Parrival） * 成交股数Sfilled
      • 机会成本Opportunity Cost = （收盘价格Pclose - 决策价格Porder） * 未成交股数Sunfilled
      • 固定费用Fixed Fees = 成交股数Sfilled * 每股手续费
  2. 交易成本分析（Trade Cost Analysis，TCA）
    1. 基础成本计量
      • 金额成本Cost(USD) = Side * (P - P*) * Shares
      • 每股成本Cost(USD/Share) = Side * (P - P*)
      • 基点成本Cost(bps) = Side * 10000bps * (P - P*) / P*

      买单Side=1，卖单Side=-1，Shares表示执行数量
      P表示执行平均价（Average Execution Price），P*表示基准参考价（Benchmark Price）
      如果将P*设为Arrival Price/VWAP/TWAP/Close Price，则原式也表示Arrival/VWAP/TWAP/MOC Cost

    2. 市场调整成本（Market-adjusted Cost）：大盘走势上涨时买入/下跌时卖出导致的额外成本
      • Market-adjusted Cost(bps) = Arrival Cost(bps) - β * Index Cost(bps)
      • Index Cost(bps) = Side * 10000bps * (Index VWAP - Index Arrival Price) / Index Arrival Price

      若指数VWAP小于指数Arrival Price，说明当日指数走低，交易股票时的股价处于高位，不利于买入，有利于卖出
      此时，买入的Index Cost为负数，Market-adjusted Cost更高；卖出的Index Cost为正数，Market-adjusted Cost更低

    3. 增加价值（Value Added）：指令的到价成本减去估计的交易前成本（Estimated Pre-trade Cost）
      • Added Value(bps) = Arrival Cost(bps) - Est. Pre-trade Cost(bps)

      增加价值用于衡量交易指令/算法的执行效果，判断交易的执行是否创造了额外业绩
      如果Added Value小于0，即到价成本低于交易前的估算成本，说明该次交易的成本高于预期，交易的业绩表现低于预期

5.交易治理

  1. 最优执行（Best Execution）：评估执行价格/成本/速度/数量、结算可能性、交易本质。

    最优交易执行不一定意味着达成最低的交易成本，而是需要综合考虑各项交易目标的平衡

  2. 决定因素：紧迫性（Urgency）；下单量（Size）；流动性/日均成交量（ADV）

  3. 合格经纪商及执行渠道：最优执行监督委员会（Best Execution Monitoring Committee）

  4. 监督执行安排的流程：声誉风险、交易错误、犯罪行为、财务稳定性

第35章：投资组合业绩评估

1.业绩评估（Performance Evaluation）

重要性：评估投资质量控制；识别投资策略有效性；了解主动管理结果；判断IPS遵循情况
业绩评估的三部分：业绩计量（Measurement）；业绩归因（Attribution）；业绩评价（Appraisal）

  1. 业绩计量（Performance Measurement）：收益计量；风险计量
    1. 收益计量：绝对收益（Absolute Return）；超额收益（Excess Return）
      • 持有期收益率（Holding Period Return，HPR）：HPR = [MV1-MV0-CF]/MV0
      • 时间加权收益率（Time-weighted Return，TWR）：TWR = (1+r1)*(1+r2)*…*(1+rn)-1
      • 货币加权收益率（Money-weighted Return，MWR）：通过现金流折现模型进行计算
    2. 风险计量：事后风险（Ex-post）；事前风险（Ex-ante）
  2. 业绩归因（Performance Attribution）：探讨历史收益/风险来源。收益归因/风险归因。
  3. 业绩评价（Performance Appraisal）：定性+定量，根据业绩计量与业绩归因综合评价。

业绩计量（Measurement）解决：投资组合的业绩表现是怎样的？——讨论各种收益率和风险要如何计量
业绩归因（Attribution）解决：投资组合的业绩是如何达成的？——讨论业绩的分解、收益与风险的来源（Source）
业绩评价（Appraisal）解决：投资组合的业绩是实力还是运气？——讨论各种评价指标：夏普、索提诺、捕获率

2.业绩归因（Performance Attribution）

  1. 有效归因流程/分类
    1. 有效归因：考虑到所有收益与风险敞口；反映决策过程；可量化主动决策；完整解释超额收益/风险
    2. 归因分类：收益/风险归因；宏观/微观归因
      • 基于收益归因（Return-based Attribution）：只使用收益率数据，适用于数据频率不足、底层数据获取难度大的情形，但准确度较低；
      • 基于持仓归因（Holding-based Attribution）：使用持仓数据，可能欠缺交易成本计量，无法反映真实的收益回报，准确度较高；
      • 基于交易归因（Transaction-based Attribution）：兼用收益率及持仓数据，准确率最高，但对数据质量和频率的要求也最高。

      宏观归因（Macro）包括：资产持有人（Sponsor）的战术决策、如何选择组合投资经理
      微观归因（Micro）包括：组合层面的业绩表现、组合投资经理的投资决策

  2. 收益归因（Return Attribution）
    1. 算数归因/几何归因：
      • 算数ER = R-B
      • 几何ER = (1+R)/(1+B)-1 = (R-B)/(1+B)
    2. 股票型基金收益归因（重点）
      1. Brinson归因模型（BHB+BF模型）：组合和基准收益率根据各行业加权平均方法计量。
        分解：行业配置效应（Allocation）；个股选择效应（Security Selection）；交互效应（Interaction）
        • 总回报G = 行业配置效应A + 个股选择效应S + 交互效应I
        • 行业配置效应
          • BF模型（Brinson-Fachler）：A = Σ（组合行业i权重-基准行业i权重）*（基准行业i收益率-基准总收益率） = Σ(Wpi-Wbi)*(Rbi-Rb)
          • BHB模型（Brinson-Hood-Beebower）：A = Σ（组合行业i权重-基准行业i权重）*基准行业i收益率 = Σ(Wpi-Wbi)*Rbi
        • 个股选择效应S = Σ基准行业i权重*（组合行业i收益率-基准行业i收益率） = ΣWbi*(Rpi-Rbi)
        • 交互效应I = Σ（组合行业i权重-基准行业i权重）*（组合行业i收益率-基准行业i收益率） = Σ(Wpi-Wbi)*(Rpi-Rbi)

        注意：BHB模型和BF模型的区别仅在于行业配置效应，BHB模型采用基准行业收益率，BF模型采用基准行业收益率与基准总收益率的差值
        若题目无明确指定，一般优先使用BF模型，BF模型可以更好的衡量行业收益率相对总体收益率的贡献

      2. 因子归因模型（Factor-based）：Cahart四因子模型Rp-Rf = Ap + RMRF + SMB + HML + WML + Ep
        • 市场因子（Return Minus Risk-free，RMRF）：表示组合收益与无风险收益之差，衡量市场系统性风险因子。
        • 规模因子（Small Minus Big，SMB）：表示小盘股（Small-cap）与大盘股（Big-cap）的投资偏好，SMB越高，组合内小盘股权重越大。
        • 价值因子（High Minus Low，HML）：表示价值股（Value）与成长股（Growth）的投资偏好，HML越高，组合内价值股权重越大。
        • 动量因子（Winner Minus Loser，WML）：衡量组合的动量股（Momentum Stocks），WML越高，组合内动量强势股权重越大。
    3. 债券型基金收益归因
      1. 基于久期的敞口分解（Exposure Decomposition-Duration Based）
      2. 基于久期的收益曲线分解（Yield Curve Decomposition-Duration Based）
      3. 基于全局定价的收益曲线分解（Yield Curve Decomposition-Full Repricing Based）
  3. 风险归因（Risk Attribution）
    相对风险（对比基准组合） 绝对风险
    自下而上（Bottom-up） 持仓对跟踪风险（Tracking Risk）的边际贡献 持仓对总风险（Total Risk）的边际贡献
    自上而下（Top-down） 归因于相对配置和证券选择 因子对总风险和特殊风险的边际贡献
    基于因子（Factor-based） 因子对跟踪风险和主动特定风险（Active Specific Risk）的边际贡献 因子对总风险和特定风险（Specific Risk）的边际贡献
  4. 宏观归因/微观归因（Macro/Micro Attribution）
    1. 宏观归因：关注基金发起人决策带来的影响。
    2. 微观归因：关注每一个基金经理的业绩状况。

3.投资基准

可分为：基于负债的基准（一般用于DB Plan整体层面，不适用于单一组合）；基于资产的基准。

  1. 基于资产的基准（Asset-based Benchmark）
    1. 绝对基准（Absolute Return Benchmark）：以绝对收益率为目标，不满足可投资性。
    2. 管理人基准（Manager Universes）：以投资经历业绩中位数为基准，仅满足可计量性。
    3. 广义市场指数基准（Broad Market Indexes，BMI）：常用，但可能与投资经理风格不契合。
    4. 投资风格指数基准（Style Indexes）：在广义指数基准上细分风格，可能缺乏合理审慎。
    5. 基于因子模型的基准（Factor-model-based）：利用因子构建的组合基准。
    6. 基于收益的基准（Return-based）：因子模型+风格指数构建，将风格指数定义为因子。
    7. 特别定制基准（Custom Security-based Benchmarks）：投资经理研究制定，缺乏透明度。
  2. 合理基准特征：
    • 清晰（Unambiguous/Clearly Identified）；
    • 可投资（Investable）；
    • 可计量（Measureable）；
    • 适当（Appropriate）；
    • 反映投资观点（Reflective of Current Investment Opinions）；
    • 事先确定（Specified in Advanced）；
    • 公允（Accountable）。
  3. 基准选择重要性：证券涉及程度；同业中位数聚集；基准原因不明确。
  4. 基准质量评估：
    • 组合总收益P = 市场基准收益M + 风格超额收益S + 主动管理收益A
    • 风格超额收益S = 风格基准收益B - 市场基准收益M
    • 主动管理收益A = 组合总收益P - 风格基准收益B
  5. 另类投资基准
    1. 对冲基金：同行对比（代表性不足/幸存者偏差/回填偏差）；夏普比率。
    2. 不动产：参考不动产指数，但流动性低、透明度不足、可能存在杠杆。
    3. 私募股权：以内部收益率（IRR）为业绩展示，并构成基准指数。
    4. 大宗商品：基于大宗商品期货投资的业绩构成基准指数。
    5. 管理型衍生品：策略差异化较大，无法找到市场指数，常用同业比较。
    6. 不良证券策略：流动性很低，难以找到基准。巴克莱困境证券指数（BDSI）。

4.业绩评价（Performance Appraisal）

  1. 区分投资技巧与运气（随机因素）：长时间的观察；足够高的置信度。
  2. 业绩评价指标
    1. 夏普比率（Sharpe Ratio）：风险收益与波动率的比值，Sa = (Ra-Rf)/σa
    2. 特雷诺比率（Treynor Ratio）：风险收益与贝塔（系统性风险）的比值，Ta = (Ra-Rf)/βa
    3. 信息比率（Information Ratio，IR）：超额收益与跟踪风险的比值，IR = (Rp-Rb)/σ(p-b)
    4. 评估比率（Appraisal Ratio，AR）：又名Treynor-Black，年化Alpha除以年化残差风险，AR = Alpha/σε

      评估比率中的超额收益Alpha和残差波动σε均来源于多因子模型的回归结果，用于衡量特定收益-风险的表现

    5. 索提诺比率（Sortino Ratio）：最小可接受超额收益除以目标半标准差
      SR = (Rp-Rt)/σd = (Portfolio Returns - Minimum Acceptable Returns)/Target Semi-deviation
    6. 捕获率（Capture Ratio，CR）：与基准相比，投资经理在市场上行/下行的相对表现
      • 上行风险（Upside Capture，UC）：当基准收益为正时，UC = 组合收益率/基准收益率
      • 下行风险（Downside Capture，DC）：当基准收益为负时，DC = 组合收益率/基准收益率

        UC>100%、DC<100%表示投资业绩超过基准；
        UC<100%、DC>100%表示投资业绩不及基准。
        当存在多期收益需要计算上行/下行风险时，应通过几何平均数取多期均值。

      • 捕获率：CR = Upside Capture（UC）/Downside Capture（DC）

        根据组合与基准收益可以画出一条曲线，横坐标为基准收益Rb，纵坐标为组合收益Rp
        CR>1表示经理业绩分布呈现正偏态（Positive Asymmetry），曲线表现为凸性（Convex）。
        CR<1表示经理业绩分布呈现负偏态（Negative Asymmetry），曲线表现为凹性（Concave）。

    7. 回撤/最大回撤（Drawdown）：一段时间的跌幅/最大跌幅。
      回撤久期（Drawdown Duration）：基金从开始亏损到累计亏损为0过程持续的时间长度。

第36章：基金经理选择

1.简介

主要层面 内容 分析方法
选择范围 合适/风格/主动或被动 是否符合IPS
投资尽职调查（Investment Due Diligence） 定量/定性（4P） 归因/捕获率/回撤；投资理念/决策过程/人员素质/监测
运营尽职调查（Operational Due Diligence） 流程/公司/投资工具/条款 后台支持/公司持续经营情况/投资工具是否合适

2.选择框架

三个层面：确定投资经理的选择范围、投资尽职调查（定量/定性）、运营尽职调查。

  1. 选择范围：根据IPS和选择原因确定，参考基准组合和风格分析。
  2. 假设检验：
    • Type I错误（Error Of Commision）：做错，做了错误的选择，雇佣无技术的经理。
    • Type II错误（Error Of Omision）：没做对，没做正确的选择，未雇佣有技术的经理。
    • Type I错误更容易计量，损失明确（Explicit Cost）；Type II错误属于机会成本（Opportunity Cost）。
  3. 定性考查：对两类错误平衡取舍，主要关注Type I（后悔厌恶/容易识别/透明度高）
  4. 影响：越有效的市场，经理获取超额收益的能力普遍越弱，则选择错误的成本越低。

3.基金经理寻找与选择

  1. 投资风格分析
    风格漂移（Style Drift）：基金经理投资风格在风险收益风格上发生了变化。
    风格分析特征：有意义（Meaningful）；准确（Accurate）；持续（Consistent）；及时（Timely）
    1. 基于收益的风格分析（Return-based，RBSA）：时间序列多元线性回归，考查投资组合收益率相对于风格指数的表现。
      优势：自上而下（Top-down），无需过多额外数据；提供不同经理/时间点的可比性；长期数据更充分。
      劣势：分析结果并非特别精确，流动性弱的证券存在过时价格（Stale Price），导致低估风险。
    2. 基于持仓的风格分析（Holding-based，HBSA）：横截面回归，关注投资组合在不同时点的实际持仓情况。
      优势：自下而上（Bottom-up），分析取决于实际持仓。
      劣势：计算难度受复杂度/透明度影响；频繁调仓影响；窗饰（Window Dressing）影响。

    窗饰/装点门面（Window Dressing）：报告期前卖出绩差股票，买入绩优股票，从而修饰持仓。

  2. 捕获率（Capture Ratio，CR）与回撤（Drawdown，DD）
    1. 捕获率：CR大于1表示经理业绩优秀，反之业绩不佳；分析与投资者IPS是否匹配。
    2. 回撤：揭示投资过程的潜在错误、风控不足或操作问题。

4.基金经理尽职调查

  1. 投资尽职调查（Investment Due Diligence）：分析基金团队的可靠性。
    关键：基金投资理念；基金管理团队的专业性。
    1. 投资理念：被动vs主动（行为/市场低效）；前提假设（持续性）；回报来源（低效能力）
    2. 人员素质：基金管理团队成员的背景和经验（专业度/人员数量/核心流失/非竞争条款）
    3. 投资决策过程：信号创造（想法）；信号捕获（执行）；组合构建；组合监测
  2. 运营尽职调查（Operational Due Diligence）：分析基金运营过程的风险。
    关键：公司后台/基础设施；第三方服务商的独立性。
    1. 投资管理公司：公司股权结构；总体资产规模（AUM）；盈亏平衡AUM
    2. 投资工具：
      • 分离式管理账户（SMA）：账户属于客户名下，按比例享有投资组合中的证券。
        优势：投资者拥有所有权；可以个性化定制；税务有效性更强；透明度高。
      • 集合投资工具（Pooled Vehicle）：共同基金/ETFs/交易所票据等，资金归入同一账户管理。
        优势：运营成本更低；追踪风险和基准偏离程度较低。
    3. 合约条款：流动性与管理费
      Billed Fee = Base Fee + Sharing*(Active Return - Base Fee), Limit to Maximum Annual Fee
